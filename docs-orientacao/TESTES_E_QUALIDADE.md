# Testes e Qualidade

## Frontend
- Priorizar testes de logica em `lib` e `services`.
- Evitar testes excessivos de markup quando nao agregarem valor.

## Lint
- Usar `npm run lint` antes de abrir PRs.
- Corrigir avisos relevantes para manter consistencia.

## Backend (quando aplicavel)
- Services devem ter testes.
- Repositorios devem ser mockados.
- Testes devem cobrir logica de negocio.
- Frameworks preferidos: JUnit e Mockito.
